Key Concepts in Web Browsing (5-7 Concepts)

Web Browser

An application that serves as the main interface between the user and the World Wide Web.

Client/Server Architecture

A distributed structure that divides workloads between providers (servers) and requesters (clients), with the browser acting as the client.

HTML (Hypertext Markup Language)

The foundational language used to describe web pages; the browser interprets the tags to render readable content.

CSS (Cascading Style Sheets)

A language used with HTML that defines the style and appearance of web pages, controlling elements like font, color, and layout.

HTTP (Hypertext Transfer Protocol)

The communication language the browser uses to send requests to a web server and receive the content-containing responses.

URL (Uniform Resource Locator)

The unique address used by the browser to locate a specific resource on a web server, including the protocol and server address.

Rendering Engine

One of the most important parts of the browser that handles scripts and markups, with examples including Gecko (Firefox) and WebKit (Chrome/Safari).
